# v1.0.2 — Watch Category column + Monitor CN-priority + Review trim

- **Date/Time:** 2026-06-27 10:30 (ต่อจาก v1.0.1)
- **Idea Ref:** ผู้ใช้สั่งตรง (4 ข้อ) — ต่อยอด [[logs/2026-06-27-v1.0.1-watch-multicat]]
- **Changes Made:**
  1. **Watch: Confidence → Category** — ตารางโชว์หมวดที่ best-of-N เลือกแทนเลข confidence. `AssetInfo` += `category`; `WatchTab` parse `j["category"]` + helper `watchCategoryLabel(flag)` (pc→"PC&Laptop"…). KPI "Avg conf." ยังอยู่ (อ่าน ocrConfidence เหมือนเดิม)
  2. **Watch monitor: CN- ชนะ Service Tag เสมอ** — `ocr_worker._score` เพิ่ม tier `is_cn` (`serial.startswith("CN-")`) **เหนือ** `is_barcode`. เลือกแบบ "CN ก่อน, ไม่มีค่อย fallback": รูปจอ → CN; รูป PC (ไม่มี CN) → Service Tag ปกติ. **verify**: รูปจอเดิม best เปลี่ยนจาก `pc→27W2NG2` (Service Tag) เป็น `monitor→CN-0JF27G-FCC00-7AM-CRPB-A04`
  3. **Review: ย่อหัว + รูปใหญ่ขึ้น** — ลบปุ่ม Load CSV / Images folder + label "CSV:(none)" (flow จริงคือ Send-to-Review); ลด margins/spacing; splitter `1:2` + `setSizes({320,760})` (เดิม 2:3 / {420,620})
  4. **OCR Single: ปุ่ม Donate → "Donate PC&Laptop"**
  - bump **1.0.2** (CMake/iss/pdf) → `AutoSuisei-Setup-1.0.2.exe` (120 MB) + `update-package.zip` (76 MB) → release **v1.0.2**
- **Technical Notes:**
  - 🔑 monitor pipeline **CN-only อยู่แล้ว** (`extract_serial_monitor` / `_pick_serial` monitor ทิ้ง 7-char). Service Tag รั่วเฉพาะ **best-of-N cross-category**: รูปจอถูกรันผ่าน `pc` pipeline ด้วย (`_valid_dell_tag` รับ Service Tag) → แก้ที่ **scoring** ไม่ใช่ pipeline
  - `is_cn` ปลอดภัย: มีแต่จอผลิต CN- (Dell PC = 7-char tag, accessory = numeric) → tier เปิดเฉพาะ read ของจอ
  - Review: `imagesFolder_` ถูกตั้งโดย `loadFromExtraction` (Send-to-Review) → ลบปุ่ม manual ได้โดยไม่กระทบ flow โหลดรูป/rename
  - Qt mnemonic: `QPushButton`/`QCheckBox` ตี `&` เป็น shortcut → ใช้ `&&` ("Donate PC&&Laptop"); แต่ `QTableWidgetItem` (cell คอลัมน์ Category) ใช้ `&` เดี่ยว (ไม่ใช่ mnemonic)
  - update-package **valid** (แก้ C++/Python logic เท่านั้น ไม่เพิ่ม pip dep)
